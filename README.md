## ダンジョン生成

### アルゴリズム

```
1. 部屋をランダム配置
2. デローニー三角形分割で全接続グラフ生成
3. MSTで最小接続（全部屋が繋がる最小の廊下）
4. MST辺の15〜20%をランダムに追加（ループ作成）
5. 特殊部屋を葉ノード（行き止まり）に割り当て
6. タイルマップに変換
```

### 部屋の種類

| 種別 | 内容 |
|------|------|
| NormalRoom | 敵配置、ドロップあり |
| TreasureRoom | 宝箱のみ、敵なし |
| StairsRoom | 下り階段、敵少なめ |
| StartRoom | スポーン地点、敵なし |
| BossRoom | ボス専用大部屋、入ると扉が閉まる |

### パラメータ

```
部屋数：5〜10部屋（フロアが深いほど増加）
部屋サイズ：最小5x5〜最大15x15タイル
廊下形状：L字（水平→垂直）、幅3タイル
```

### タイルタイプ

```
FLOOR(0)      歩行可能
WALL(1)       衝突あり
WALL_EDGE(2)  壁装飾用（衝突あり）
```

### フォグオブウォー

```
HIDDEN    未探索（真っ黒）
REVEALED  探索済み（暗く表示）
VISIBLE   現在見えている
```

プレイヤー周囲の半径R内を毎フレームVISIBLEに更新。  
範囲外のVISIBLEはREVEALEDに落とす。  
実装：タイルごとのアルファ制御（軽量）

## 敵システム

### 敵タイプ

| タイプ | 行動パターン |
|--------|-------------|
| Chaser | プレイヤーに直進 |
| Shooter | 距離を保ちながら弾を撃つ |
| Rusher | 遠距離は遅く、近距離で急加速 |
| Splitter | 倒すと小型に分裂 |
| Summoner | 一定間隔で雑魚を召喚 |

### フロア深度と出現テーブル

```
F1-20    Chaser, Shooter
F21-40   Chaser, Shooter, Rusher
F41-60   Chaser, Shooter, Rusher, Splitter
F61-80   全タイプ
F81-100  全タイプ + エリート版（ステータス倍率増）
```

### 敵配置ロジック

```
配置対象：NormalRoomのみ
敵数：floor(部屋面積 / 20) ± 1（最小1）
壁際・廊下入口付近は配置除外
敵種別：フロア深度の重みテーブルから抽選
```

### アクティブ化

```
EnemyGroup {
  roomId
  enemies: Enemy[]
  isActive: boolean  // プレイヤーが部屋に入ったらtrue
}
```

未アクティブの敵はupdate処理をスキップ。

### 更新頻度（パフォーマンス対策）

```
画面内の敵     毎フレーム更新
画面外の敵     数フレームに1回
未アクティブ   更新しない
```

### 移動AI

ベクトル追跡（Steering Behavior）+ 壁回避のみA*の折衷方式。

## 属性システム

| 属性 | 効果 |
|------|------|
| Fire | DoT（燃焼、毎秒ダメージ3〜5秒） |
| Ice | 移動速度低下・スタック重複で凍結 |
| Thunder | スタン（一瞬動きが止まる） |
| Poison | 防御力を一時的に低下 |
| None | 純粋魔力ダメージ |

### 属性倍率

```
弱点属性  1.5倍ダメージ
耐性属性  0.5倍ダメージ
```

敵は `resistance`（耐性）と `weakness`（弱点）を1つずつ持つ。

## ワンドシステム

### データ構造

```
Wand {
  name          // 例：「炎のワンド」「古びた雷杖」
  attribute     // Fire / Ice / Thunder / Poison / None
  rarity        // Common / Uncommon / Rare / Epic / Legendary
  stats {
    damage
    fireRate
    projectileSpeed
    piercing      // 貫通数
  }
  specialEffect  // Rare以上に付与
}
```

### レアリティと特殊効果

| レアリティ | 特殊効果数 |
|-----------|----------|
| Common | なし |
| Uncommon | なし |
| Rare | 1つ |
| Epic | 2つ |
| Legendary | 3つ |

### 特殊効果一覧

| 効果 | 内容 |
|------|------|
| Multishot | 弾が複数方向に出る |
| Homing | 弾が敵に追尾する |
| Explosion | 着弾時に範囲爆発 |
| Chain | 敵に当たると別の敵に連鎖 |
| Lifesteal | ダメージの一部をHPに変換 |

### ドロップ

- 敵を倒したときランダムドロップ
- 宝箱（TreasureRoom）からドロップ
- ドロップ時に属性・レアリティをランダム決定

---

## 成長システム

### 経験値・レベルアップ

- 敵を倒すと経験値を取得
- 一定値に達するとレベルアップ
- レベルアップ時にSP（ステータスポイント）を1点取得
- SPを好きなステータスに振り分け

### ステータス（SPECIAL参考）

| 略称 | 名称 | 効果 |
|------|------|------|
| P | Power | ワンドのダメージ倍率 |
| I | Insight | 視野範囲、ミニマップ精度 |
| V | Vitality | 最大HP、HP回復速度 |
| F | Fortune | ドロップ率、レアリティ補正 |
| A | Arcana | 経験値取得量、属性効果強化 |
| S | Swiftness | 移動速度、ワンド連射速度 |
| T | Fate | クリティカル率、特殊効果発動率 |

各ステータスの上限：20

## ボス一覧

| フロア | 名称 | 属性 | 特徴 |
|--------|------|------|------|
| F10 | 炎の魔獣 | Fire | 突進 + 炎の壁設置 |
| F20 | 氷の巨人 | Ice | 範囲凍結 + 氷柱召喚 |
| F30 | 雷の鳥 | Thunder | 高速移動 + 広範囲雷撃 |
| F40 | 毒の蜘蛛 | Poison | 毒沼設置 + 雑魚召喚 |
| F50 | 無属性の騎士 | None | 全属性耐性 + 反射弾 |
| F60 | 炎氷の双子 | Fire/Ice | 2体同時、片方撃破でもう片方強化 |
| F70 | 雷の魔導士 | Thunder | プレイヤーと同じ自動射撃、高火力 |
| F80 | 毒の樹 | Poison | 移動しない、範囲攻撃のみ、高HP |
| F90 | 虚無の影 | None | 分身生成、本体を探して撃破 |
| F100 | 深淵の王 | 全属性 | フェーズ制、全ボスパターン複合 |

### ラスボス（F100）フェーズ制

```
フェーズ1  HP 100〜60%  通常攻撃 + 全属性弾
フェーズ2  HP 60〜30%   移動速度上昇 + 範囲攻撃追加
フェーズ3  HP 30〜0%    フィールド全体攻撃、逃げ場が限られる
```

### ボスフロア構造

```
専用の大部屋（通常より広い）
入ったら扉が閉まる
撃破後に下り階段が出現
ドロップのレアリティが高め
```

## 技術設計メモ

### 弾丸システム

```
Projectile {
  position, velocity, lifetime
  damage, piercing, bounceCount
  attribute
  onHit: HitEffect[]  // 貫通、爆発、毒など合成可能
}
```

### 自動エイム

最近傍の敵に向けてワンドが自動発射。  
エイム戦略はプラグイン化しておくと拡張しやすい。

### タイルマップ変換（Phaser 3）

```javascript
const map = this.make.tilemap({ data: dungeonGrid, ... })
const tiles = map.addTilesetImage(...)
const layer = map.createLayer(0, tiles)
layer.setCollisionByProperty({ collides: true })
```

### スプライト方針

アセット管理を不要にするため、Canvasで図形を組み合わせたドット絵風スプライトを使用。
